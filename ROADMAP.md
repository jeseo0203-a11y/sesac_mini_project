# 취미로(Chwimiro) 취미 추천 AI 서비스 개발 로드맵 (ROADMAP.md)

본 문서는 "취미로" 취미 추천 AI 에이전트 서비스 개발을 위한 단계별 마일스톤과 세부 개발 계획을 정의합니다. `AGENTS.md` 및 `SKILL.md`를 바탕으로 작성되었습니다.

---

## 📌 서비스 개요
"취미로"는 구조화 설문(9축) 및 적응형 대화를 통해 사용자의 상태와 제약을 파악하고, 최적의 취미 3~5개를 처방하는 AI 에이전트 서비스입니다. 
- **핵심 가치**: 사용자 맞춤형 안정적 취미 추천 + "의외의 발견(Serendipity)" 추천 제공
- **주요 페르소나**: 26세 여성 스타트업 마케터 (번아웃 상태, 저비용/저시간 실내 개인 활동 선호)

---

## 🛠️ 기술 스택 및 아키텍처
- **Front-end**: Next.js + Tailwind CSS (Vercel 배포)
- **Back-end**: FastAPI (Railway/Render 배포)
- **Vector DB**: ChromaDB (취미 RAG설명 임베딩 검색)
- **RDB**: Supabase (Postgres)
- **AI Core**: OpenAI SDK (GPT-5.4-mini) & OpenAI Embedding API

## ⚠️ 핵심 제약 사항 & 구현 원칙
1. **의외 추천 보장**: 모든 처방 결과에는 반드시 `recommendation_type = "의외"` 카드가 1개 포함되어야 합니다.
2. **LLM 임의 생성 제한**: `초기비용`, `지속비용`, `safety_notes(안전)`는 LLM이 생성하지 못하도록 차단하고, DB 데이터만 제공합니다.
3. **의학적 표현 금지**: "우울증", "진단" 등의 표현 대신 "오늘의 컨디션", "에너지 상태" 등으로 순화합니다.
4. **API Key 보안**: 어떠한 경우에도 프론트엔드에 API Key가 노출되어서는 안 되며, 모든 LLM/DB 호출은 백엔드를 거칩니다.
